博客平台选择

wordpress/hexo/hugo的比较

动态博客 vs 静态博客

动态博客: 如wordpress, typecho等

  • 优点(主要以wordpress为例):

    • 上手简单, 安装可能更容易

    • 功能全, 开箱即用:

      • 有在线写作界面, 有网就能写
      • 有后台管理界面, 可以可视化管理, 容易上手, 更专注创作
      • 有评论系统
      • 有阅读统计系统

    • 有很多主题, 但很多wordpres主题不适合国内使用, 找主题比较折腾

    • 有大量插件可以增强功能

  • 缺点:

    • 需要购买和配置服务器, 有些服务商会提供配置好的环境
    • 需要管理大量插件, 插件太多可能引起性能问题
    • 需要操心安全问题
    • 相对于静态博客, 好看的主题比较少

  • 使用动态博客你可能需要折腾的东西

    • 找主题, 修改定制主题
    • 找编辑器插件(如果你想使用markdown编辑)
    • 安装各种插件, 安全插件, 公式插件等等

  • 可能的硬伤(也许能通过其他方式来绕过):

    • 在线编辑体验不够好, 比不上本地编辑器(如typora)的体验
      • 曲线救国: 如 mark2press 这样的工具, 可以实现本地编辑然后发布到wordpress
    • 同样是由于在线写作发布的问题, 文章不方便同时发布到其他平台
      • 曲线救国: 使用如OpenWrite这样的多平台发布工具

静态博客: 如hexo, hugo等

  • 优点(主要以hexo为例):

    • 安全: 纯静态页面, 无后台管理页面, 基本不用考虑站点安全问题
    • 数据完全自己掌握: 自己会很清楚文件是怎样保存的
    • 成本低: 网上有很多方法实现零成本部署, 如 github page 等
    • 最佳的本地编辑体验: 可以在本地使用自己喜欢的markdown编辑器, 可以拥有最佳的编辑体验
    • 方便发布到多个平台
    • 后期维护成本极低, 后期更省心.

  • 缺点:

    • 前期学习曲线稍高(因主题而定, 一些好用的出题还是很容易上手的)
    • 比动态博客更加折腾, 想折腾好难度还比较高, 见下方
    • 发布稍微麻烦, 需要在本地编辑好在通过一些方法部署到网络上, 但打通工作流后就可以一键部署
    • 无法在线编辑(现在也有一些方法实现在线编辑)

  • 使用静态博客你可能需要折腾的东西:

    • 访问量统计

    • 评论系统

    • 搜索功能

    • 发布流程

    • SEO

    • 图床管理问题

    • 性能优化

  • 硬伤:

    • 无明显硬伤, 主要是如果要添加丰富的功能会比较折腾

动态和静态博客如何选择?

可以从一定要有的点和不能忍受的缺点出发做选择

  • 想低成本的使用 -> 选静态, 网上有很多方法能实现0成本
  • 担心安全问题 -> 选静态
  • 想对数据组织有非常高的管控程度/了解程度 -> 选静态, 文件结构一目了然, 管理轻松
  • 对编辑体验有极高需求 -> 选静态, 可以使用本地编辑器
  • 想在线编辑,管理 -> 选动态
  • 想开箱即用, 不想太折腾 -> 选动态

动态博客想尝鲜的话, wordpress.com 甚至可以免费使用.

hugo和hexo如何选择?

选择hexo是因为遇到了一个喜欢的主题. 感觉hexo更容易上手一些. 安装插件比较方便, 可以通过插件的方式来增加一些功能.

hexo可能在文章过多的时候可能存在构建较慢的性能问题, 但网上已经看到了一些基于hexo发布了500篇以上的博客, 所以前期这个性能问题应该不太重要, 后面碰到再说.
